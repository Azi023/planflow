## 2. KPI METRICS — INPUT vs. CALCULATED

### INPUT METRICS (entered by planner):
- Campaign Budget (LKR) — total with mgmt fee noted
- Campaign Period — free text
- Platform — dropdown/text
- Targeting Audience / Targeting Criteria — free text block
- Estimated Audience Size — text estimate
- Channels / Objectives — text
- Budget (LKR) — per-row allocation
- Percentage (%) — allocation split (sometimes calculated, sometimes entered)
- CPM range (Values are in LKR) — manual entry of benchmark
- CPC, CPL, CPV — benchmark rates entered manually
- Range Given by platforms CPM/CPL — platform benchmark rates
- Ad Type & No of content pieces — free text
- Country — text
- Buy Type — text (Auction)
- Frequency — text (e.g., "2+")
- Creative / Assets / Content — text description

### CALCULATED/OUTPUT METRICS (derived):
- **Reach** — derived from Budget / CPM × 1000 (but entered as estimates, not live formulas in most sheets)
- **Impressions** — Reach × Frequency
- **Video Views** — from CPV calculation
- **Leads** — from CPL calculation
- **Total Budget (LKR)** — `=Media Spend × 1.15` (15% management fee added)
- **Total Budget With Management Fee (LKR)** — `=Amount Spend + Mgt Fee + ASP Fee`
- **Mgmt Fee amount** — `=Media Spend × 0.15` or `× 0.12`
- **ASP Fee** — `=Media Spend × 0.0625`
- **TOTAL MEDIA SPEND** — `=SUM()` of per-row budgets

### FORMULA PATTERNS OBSERVED:
- Most numeric outputs appear to be **manually typed estimated ranges** (e.g., "3M", "1.2M"), NOT live formula-calculated
- Total/TOTAL rows use `=SUM()` formulas
- Management fee rows use multiplication: `=B_n * 0.15`
- The "Percentage (%)" column uses: `=[Budget cell]/[Total Budget cell]` in some files
- TOTAL MEDIA SPEND LKR conversion = USD total × exchange rate (hardcoded rate)
- No complex nested formulas observed — predominantly arithmetic calculations
- No `#REF!` or visible formula errors detected

## 7. BUDGET STRUCTURE

### Split Logic:
**By Platform:** Multi-platform files split rows by platform (Meta/GDN/YouTube/TikTok) with separate budget per platform row. In newer template F files, each platform gets its own sheet tab.

**By Objective/Channel:** Within same platform, rows split by objective (e.g., Reach 57% + Leads 43%, or Awareness + Traffic). The "Channels/Objectives" column drives row splitting.

**By Audience:** In remittance/multi-audience files, rows split by audience name (Overseas audience + Local audience), each with own budget and objective.

**By Ad Type:** Some splits go to ad format level (Main Video row vs. Static post row).

### Management Fee Patterns:
| Client / File Type | Management Fee | Notes |
|---|---|---|
| Peoples Bank (most files) | **15%** | "15%" shown in Mgmt Fee column; calculated as Media Spend × 0.15 |
| Peoples Bank (old USD template) | **15%** | Same |
| Peoples Bank FB Likes | **15%** | "Inclusive of 15% Agency commission" |
| Fonterra / Anchor Pedia Pro | **15%** | "Agency Fee (15%)" |
| Anchor Milk / Anchor Newdale | **12% + ASP 6.25%** | Two separate fee lines: Mgt Fee (12%) + ASP Fee (6.25%) |
| Anchor Milk CP Campaign | Combined into "Total Budget With Mgt & ASP Fee" | |

### Currency Handling:
- **LKR (Sri Lankan Rupees)** — primary currency for almost all Peoples Bank files
- **USD** — used for TikTok campaigns (TikTok billed in USD), some older Peoples Bank files (Credit Card Offers revised, Christmas Meta tab)
- USD → LKR conversion: Hardcoded exchange rate (approximately LKR 312/USD seen in one file: "$ 3,030.30" → "$ 258,750.00 LKR" suggests ~LKR 312 × 830 = complex)
- **"Including management fee"** notation on Campaign Budget cell is standard across Template A
- Media Spend is NET (without fee); Total Budget is GROSS (with fee added)
